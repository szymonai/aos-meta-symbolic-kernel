# Regulatory Readiness, Not Compliance

This repository is organized to support future evidence work. It does not claim
EU AI Act compliance, MDR/MDSW compliance, CE marking, ISO certification,
production readiness, clinical evaluation completion, or post-market readiness.

| Framework | Current evidence | Missing | Next action |
| --- | --- | --- | --- |
| EU AI Act | Public boundary claims, audit concept, deterministic demo gate | Risk classification, QMS evidence, human oversight process, data governance, logging policy, robustness evidence | Build an AI Act evidence map and risk-management file outside the public demonstrator |
| MDR/MDSW | Radiology scenario is documented as non-product reference only | Intended medical purpose, clinical evaluation, software classification, PMS/PMCF, technical documentation | Keep public repo non-medical-device; prepare regulatory strategy before any product claim |
| ISO 13485 | Documentation discipline and non-redistribution boundary | Quality management system, design controls, supplier controls, CAPA, document control | Establish QMS separately before regulated development |
| ISO 14971 | PASS/WARN/BLOCK can support risk-control discussion | Hazard analysis, risk acceptability, residual-risk evaluation, risk-control verification | Create a risk-management file linked to specific intended use |
| IEC 62304 | Public code has tests and CI | Software safety class, architecture, SOUP analysis, requirements traceability, release process | Define software lifecycle process for controlled development |
| IEC 62366 | Human review/escalation is documented conceptually | Usability engineering file, user profiles, use-error analysis, formative/summative studies | Define human-in-the-loop workflow and usability plan |
| ISO 27001 | SECURITY.md and secret scanning/push protection support basic hygiene | ISMS scope, risk treatment, access controls, supplier controls, audit process | Manage security program outside this public demonstrator |
| GDPR | No patient data redistributed in repo | DPIA, lawful basis, DPA, retention policy, data-subject process, cross-border transfer review | Keep public repo data-free; review data processing with counsel |
| Clinical evaluation | No clinical claim; only aggregate offline evidence | Clinical evaluation plan, qualified clinical input, representative datasets, statistical analysis plan | Treat local metrics as preliminary engineering evidence only |
| Post-market monitoring | No product claim | PMS/PMCF plan, complaint handling, incident reporting, monitoring metrics | Required only for future regulated product pathway |
